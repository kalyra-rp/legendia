Roadmap de développement · v.1

Construire Legendia, couche par couche

Une feuille de route pensée pour le « deploy early » : en ligne dès le jour 1, partageable à tout moment. Chaque phase marche seule avant la suivante. Coche au fur et à mesure.

Étape 0 · a

La stack

Tout ce qu'il te faut, et rien de plus. Du moderne, du bien documenté, du gratuit pour démarrer.

▲Next.js framework ◆Vercel hébergement ▣Supabase base + auth 🔗GitHub code + versions 🎨Tailwind styles ✦Claude API le MJ

💡 Le trio qui se parle tout seul

GitHub + Vercel + Supabase forment une chaîne automatique : tu pushes ton code sur GitHub → Vercel le déploie tout seul en quelques secondes → ton site en ligne lit/écrit dans Supabase. Une fois branché, tu ne penses plus à « déployer » : tu codes, tu pushes, c'est en ligne.

Étape 0 · b

Le socle déployé

L'objectif du tout premier jour : un site vide mais EN LIGNE, et la chaîne auto-déploiement qui tourne. Avant toute fonctionnalité.

Créer le repo GitHub
Public ou privé, peu importe. Un README qui dit ce qu'est Legendia.
Initialiser le projet Next.js
npx create-next-app avec Tailwind coché. Pousser le tout sur GitHub.
Brancher Vercel sur le repo
Importer le repo dans Vercel → premier déploiement automatique. Tu as déjà une URL en ligne.
Créer le projet Supabase
Noter l'URL du projet et la clé publique (anon key).
Connecter Supabase à Next.js
Les clés dans les variables d'environnement Vercel (jamais dans le code !). Installer le client Supabase.
Page d'accueil « Le Seuil » minimale
Un titre, le logo, « bientôt ». Juste pour voir quelque chose de joli en ligne et le montrer aux amis.

⚠️ La règle d'or des secrets

Tes clés (Supabase, plus tard Claude) ne vont JAMAIS dans le code poussé sur GitHub. Toujours dans les variables d'environnement de Vercel. Un secret poussé par erreur sur un repo public, c'est la porte ouverte. Prends le réflexe dès maintenant.

Phase I

Compte & accès

Le verrou d'entrée : connexion Discord, et vérification d'appartenance au serveur. La fondation de toute la modération.

But : un visiteur peut se connecter avec Discord, et seuls les membres du serveur entrent. Les autres voient une porte fermée.
🚀 Livrable : connexion fonctionnelle en ligne
Activer l'auth Discord dans Supabase
Supabase gère les connexions « OAuth ». Créer une appli Discord développeur, brancher les clés.
Bouton « Se connecter avec Discord »
Sur Le Seuil. Au clic, l'utilisateur s'authentifie via Discord et revient connecté.
Vérifier l'appartenance au serveur
À la connexion, demander à Discord si la personne est bien membre du serveur. Sinon → accès refusé, message doux.
Table profils + rôle
Créer chaque membre en base avec une colonne rôle (membre / modo / admin). Toi en admin.
Page protégée de test
Une page « tu es bien connecté·e, [pseudo] » visible seulement une fois loggé. La preuve que le verrou marche.

💡 Pourquoi commencer par là

L'auth est le socle de tout le reste : pas de perso, pas de scène, pas de board sans savoir qui est connecté et avec quel rôle. C'est moins gratifiant visuellement, mais c'est la fondation. Une fois debout, tout s'appuie dessus.

Phase II

Le Hall & les univers

Le membre connecté arrive quelque part, et découvre l'univers Saint-Pierre. Le squelette de navigation.

But : après connexion, j'arrive au Hall, je vois l'univers Saint-Pierre et je peux consulter sa bible (le lore) dans le site.
🚀 Livrable : navigation + lore consultable en ligne
Layout général + navigation
La structure des onglets (Hall, Univers, etc.), le menu, l'identité visuelle de Legendia.
Table univers
Créer Saint-Pierre en base : nom, description, statut (ouvert/fermé).
L'onglet « Les Univers »
Liste les univers ouverts. Pour l'instant, juste Saint-Pierre.
Intégrer la bible (le lore)
La bible navigable qu'on a écrite devient une page du site. Tu l'as déjà — c'est du copier-intégrer.
Rejoindre un univers
Un bouton « rejoindre Saint-Pierre » qui lie le membre à l'univers en base.
Phase III

Personnages & fiches

Le cœur du RP : créer son personnage, le voir, l'éditer. Et le board de validation pour toi.

But : je crée un personnage via un formulaire, il part en validation, tu l'approuves, il devient jouable et visible.
🚀 Livrable : création + validation de persos en ligne
Table personnages
Nom, âge, quartier, métier, histoire, avatar, statut (brouillon/en attente/validé), lié au joueur et à l'univers.
Formulaire de création (Le Carnet)
Guidé, chaleureux, calqué sur le chapitre VI de la bible (quartier, métier, failles…).
Affichage d'une fiche
La fiche jolie, lisible, au format de la bible. Consultable par les autres.
Board admin : valider les fiches
Ta vue « fiches en attente » → approuver / renvoyer en correction. Le premier morceau du board admin.
Upload d'avatar
Stockage d'images via Supabase Storage. Optionnel au début (on peut mettre des initiales colorées).
Phase IV

La Ville & les scènes

Le RP devient jouable : les lieux, les scènes, écrire en tant que son personnage.

But : j'ouvre une scène dans un lieu, j'y écris en tant que mon perso, d'autres me rejoignent et répondent. Le RP, enfin.
🚀 Livrable : le RP fonctionne entre joueurs
Tables lieux et scènes
Les lieux de la bible (café, plage, marché…). Une scène = un lieu + des participants + des messages.
Liste des lieux / la Ville
L'onglet Ville : voir les lieux, les scènes en cours. (Le plan interactif joli viendra en phase VII.)
Ouvrir / rejoindre une scène
Créer une scène dans un lieu, choisir le statut (libre / animée), y entrer avec un de ses persos.
Écrire en tant que son perso
Le message s'affiche sous le nom + avatar du personnage. L'auteur (joueur/perso) est stocké en base.
Le fil de scène en temps réel
Les nouveaux messages apparaissent sans recharger. Supabase Realtime fait ça nativement.
Phase V

Le MJ entre en scène

L'âme de Legendia : les deux boutons (intervenir / ambiance), nourris par Claude et la bible.

But : dans une scène, je clique « faire intervenir » → un PNJ répond, joué par Claude, cohérent avec sa fiche et le lieu.
🚀 Livrable : le MJ joue les PNJ en ligne
Table pnj
Les piliers de la bible (Magloire, Rosa…) avec leur fiche, surtout la « voix / parler ».
Brancher Claude API (côté serveur)
Un point d'entrée sécurisé qui appelle Claude. Clé dans les variables d'env, jamais exposée au navigateur.
Bouton « ✦ Faire intervenir » + fenêtre
L'option A : choisir qui (pilier/figurant) + quoi. Construit le contexte (fiche + scène + lore) et appelle Claude.
Bouton « 🌿 Demander une ambiance »
Pose une atmosphère sans personnage, appuyée sur le lieu + le mois en cours.
Les garde-fous
Délai de politesse, bouton pause, ton doux. Écrire le « prompt système » du MJ à partir de sa fiche de conception.

💡 C'est ici que ta bible paie

Tout le lore qu'on a écrit (lieux, PNJ, voix, calendrier) devient le carburant envoyé à Claude. Plus la bible est riche, plus le MJ est juste. Tu récoltes ce que tu as semé ces dernières heures.

Phase VI

Le Beffroi & la vie

Ce qui fait revenir : les notifications, le rythme mensuel, les annonces.

But : je reçois des notifs (« à toi d'écrire », « le MJ a ouvert le mois »), et le MJ rythme la vie au mois.
🚀 Livrable : la plateforme vit toute seule
Table notifications + Le Beffroi
L'onglet qui regroupe : à toi d'écrire, annonces, le MJ a parlé, relations.
Générer les notifs « à toi d'écrire »
Quand quelqu'un répond dans ma scène, une notif apparaît au Beffroi.
Board admin : annonces
Publier une annonce → elle s'affiche au Beffroi de tous. Éditer la charte.
Le rythme mensuel du MJ
Ouverture / clôture de mois automatique ou déclenchée par toi. Le curseur de présence par univers.
Notifications push (PWA)
Plus tard : les notifs sur le téléphone. Nécessite l'installation de la PWA. Optionnel mais puissant.
Phase VII

Polish & ouverture

Les finitions qui transforment un outil en expérience, puis l'ouverture aux premiers joueurs.

But : tout est beau, fluide, installable. On ouvre à un petit groupe test, puis à la communauté.
🚀 Livrable : Legendia ouvre ses portes
Le plan interactif de la Ville
La carte cliquable des lieux (le joli, version aboutie de l'onglet Ville).
Le Carnet complet
Relations entre persos, journal de vie, domicile. La mémoire de chaque personnage.
PWA installable
Manifest + icône → Legendia s'installe sur l'écran d'accueil comme une app.
Tests avec quelques amis
Inviter 3-4 personnes de confiance, jouer une vraie scène, noter ce qui coince.
Ouverture douce
Ouvrir le Discord-vitrine, accueillir les premiers vrais joueurs. Legendia est née.
Méthode

Conseils de route

Quelques principes pour ne pas te décourager et avancer sereinement.

🚀 Pushe souvent, pushe petit

Chaque petit bout qui marche → push → déployé. Tu vois ton progrès en ligne en continu, et si quelque chose casse, c'est facile de retrouver quand. Ne garde jamais des jours de travail non poussé.

🧱 Une couche à la fois, finie avant la suivante

Résiste à l'envie de tout faire en parallèle. Chaque phase doit marcher seule avant d'attaquer la prochaine. Un perso qu'on crée mais qu'on ne peut pas voir, c'est frustrant — finis le bout.

🎨 Le moche-qui-marche d'abord, le beau ensuite

Aux phases I–VI, vise le fonctionnel, même laid. Le polish (phase VII) viendra. Un bouton gris qui marche vaut mieux qu'un bouton magnifique qui n'existe pas encore.

🆘 Quand tu bloques

Tu vas bloquer, c'est normal et sain. Reviens me voir avec l'erreur précise, ou le bout de code qui coince. On débogue ensemble. Ne reste jamais coincée seule des heures — c'est là qu'on se décourage.

📚 En parallèle : ton apprentissage

Ce projet EST ton école Next.js + Supabase. Chaque phase t'apprend une brique fondamentale (auth, base de données, temps réel, API…). Tu apprends en construisant quelque chose que tu aimes — la meilleure façon.
